# Plan: Implement Closure Variable Capture Filtering

## Goal
Enable ExifTool compilation by filtering closure captures to only include variables actually referenced in the closure body, reducing the 154 captured variables down to just the few actually used.

## Current Status

### Working:
- ✅ VariableReferenceCollector properly identifies which variables are used
- ✅ Name-based lookup in EmitVariable.java (looks up by name instead of using baked-in AST indices)
- ✅ Defensive measures prevent infinite loops
- ✅ All unit tests pass with filtering disabled

### Blocker:
- ❌ VerifyError when filtering is enabled: "Type null (current frame, locals[18]) is not assignable to double"
- This occurs in generated closure apply() methods
- Happens even in feature.pm during startup, not just ExifTool

## Root Cause Analysis

When we filter variables and preserve original indices:

1. **newEnv array has gaps**: If we filter and keep only variables at indices 5, 10, 20, the newEnv is [null, null, null, null, null, "var1", null, null, null, null, "var2", ...]

2. **Constructor creates fields with gaps**: EmitterMethodCreator creates instance fields for all positions in newEnv, including nulls

3. **apply() method initializes locals**: Lines 539-551 in EmitterMethodCreator.java copy from fields to local slots, putting null in gap slots

4. **Code tries to use gap slots**: Somewhere in the generated bytecode, code tries to load a double from a slot that's null

## Problem Hypothesis

The VariableReferenceCollector may be missing some variable references:
- Variables in string interpolation: `"$var"`
- Variables in eval strings
- Implicit variables like `$_` in grep/map
- Variables accessed via symbolic references
- Variables in nested closures

Or there's an issue with how temporary locals are allocated - they might overlap with the gaps in filtered captures.

## Implementation Plan

### Phase 1: Improve Variable Detection (2-3 hours)

1. **Audit VariableReferenceCollector**
   - Check each visit method handles all variable reference cases
   - Add visit methods for StringNode if missing (string interpolation)
   - Handle implicit $_ in grep/map/sort blocks
   - Test with --debug logging to see what variables are detected

2. **Add conservative fallbacks**
   - If a closure body contains eval, disable filtering for that closure
   - If string interpolation detected, scan the string for variable references
   - Consider keeping all variables if uncertainty detected

3. **Test incrementally**
   ```bash

   ./jperl -E 'my ($a,$b,$c)=(1,2,3); sub{$a+$c}->()'

   # Test 2: String interpolation
   ./jperl -E 'my ($a,$b)=(1,2); sub{"value=$a"}->()'

   ```

### Phase 2: Fix Local Slot Allocation (1-2 hours)

The issue might be with how EmitterMethodCreator allocates temporary local variables:

1. **Check allocateLocalVariable()** in ScopedSymbolTable
   - When we set `index = maxIndex + 1`, new temps start after the last filtered variable
   - But if maxIndex is 20 and we only have 3 filtered variables, we're wasting slots 4-19
   - The JVM verifier might not like sparse local variable tables

2. **Solution**: Compact the indices
   - Instead of preserving original indices, assign sequential indices 3, 4, 5... (after this/@_/wantarray)
   - Update the constructor invocation code to load from correct outer indices
   - This creates a dense local variable table without gaps

### Phase 3: Constructor Parameter Mapping (2-3 hours)

Currently the constructor invocation in EmitSubroutine.java (lines 254-260) assumes:
```java
for (Integer currentIndex : visibleVariables.keySet()) {
    if (newIndex >= skipVariables) {
        mv.visitVarInsn(Opcodes.ALOAD, currentIndex); // Load from outer context
    }
    newIndex++;
}
```

With filtering + compacted indices, we need:
- Load from outer context at index `currentIndex`
- Store in closure at compacted index `compactedIndex`

**Implementation**:
```java
int compactedIndex = skipVariables;
for (Integer outerIndex : visibleVariables.keySet()) {
    mv.visitVarInsn(Opcodes.ALOAD, outerIndex); // Load from outer context
    // Constructor parameter at position (compactedIndex - skipVariables)
    compactedIndex++;
}
```

Then when adding to new symbol table, assign compacted indices:
```java
int compactedIndex = skipVariables;
for (SymbolTable.SymbolEntry entry : visibleVariables.values()) {
    currentScope.variableIndex.put(
        entry.name(),
        new SymbolTable.SymbolEntry(compactedIndex++, ...)
    );
}
```

### Phase 5: Edge Cases

1. **Nested closures**
   - A closure inside a filtered closure
   - Does the inner closure see the filtered or original variables?

2. **Closure modifying captured variables**
   - `my $x = 0; sub { $x++ }`
   - Ensure the reference is correctly captured

3. **Package subs**
   - Already handled: `isPackageSub` check skips filtering for package subs

4. **'our sub' declarations**
   - Already handled: filtered out from captures
